0.1 Introduction
In Part 1, we treat a grayscale image as a 2D matrix of intensities. All operations here are
pixel-wise (a.k.a. point operations):
output(x, y) = f (input(x, y))
That is, each pixel is processed independently, without using neighbors.
0.2 Arithmetic Operations on Images
0.2.1 Why Arithmetic Operations? (Conceptual Motivation)
Arithmetic operations can be used to:
• adjust global brightness/contrast (e.g., blending, scaling),
• emphasize differences between two images (subtraction, absolute difference),
• combine information from multiple sources (addition, weighted sum),
• normalize by a reference image or illumination field (division).
0.2.2 Standard Practical Workflow (Step-by-step)
For most arithmetic tasks in this lab, follow this workflow:
1. Read image(s) → grayscale.
2. Treat image as a 2D array of intensities.
3. Convert to float in [0, 1] for safe math (NumPy version).
4. Apply the arithmetic transform (possibly with parameters).
5. Clip/normalize results into a valid displayable range.
6. Convert back to uint8 and display/save.
0.2.3 Gamma Correction (Power-law Transform)
Theory (Short): Gamma correction is a non-linear intensity transform:
s = rγ, r ∈ [0, 1]
Interpretation:
• γ < 1 expands dark tones (brightens shadows) and compresses highlights.
• γ > 1 compresses shadows and expands highlights (overall darker).
• γ = 1 leaves the image unchanged.
3
When to use:
• Improve visibility in underexposed images ( γ < 1).
• Reduce blown-out highlights ( γ > 1).
• As preprocessing before thresholding/segmentation to separate modes in histogram.
Caution: If the image has strong noise in dark regions, using γ < 1 can amplify that
noise.
NumPy Implementation
def gamma_correction_np ( img_u8 : np. ndarray , gamma : float ) -> np. ndarray
:
"""
Gamma correction : s = r^gamma (r in [0 ,1]) .
gamma < 1: brighten shadows ; gamma > 1: darken highlights .
"""
if gamma <= 0:
raise ValueError ( "gamma must be > 0")
r = to_float01 ( img_u8 )
s = np. power (r, gamma )
return to_uint8 (s)

0.2.4 Addition / Subtraction / Multiplication / Division
Theory (Short): These are element-wise operations. For two images A and B:
Sadd = αA + βB, S sub = αA − βB, S mul = A ⊙ B, S div = A ⊘ (B + ϵ)
Practical meanings:
• Addition: blending / increasing brightness.
• Subtraction: change detection; highlight differences.
4
• Multiplication: emphasize common bright/dark regions (acts like masking in continuous
intensity).
• Division: normalize by a reference (e.g., illumination correction); can strongly amplify
values.

0.3 Logical Operations on Images (AND / OR / XOR)
0.3.1 Motivation
Logical operations are most meaningful for binary masks :
• combine multiple masks (OR),
• keep only the intersection of conditions (AND),
• detect disagreements/differences between masks (XOR),
• extract Region of Interest (ROI) by masking the original image.
0.3.2 Practical Workflow
1. Produce binary images (manual threshold or Otsu).
2. Apply AND/OR/XOR on binary results.
3. Optionally apply the binary mask to the grayscale image to extract ROI.

0.4 Otsu Automatic Thresholding
0.4.1 Motivation
Manual thresholding requires choosing a threshold by trial-and-error. Otsu automatically
selects a global threshold based on the image histogram, aiming to separate the image into
two classes (background vs foreground).
0.4.2 Key Idea (Short Theory)
Otsu chooses threshold t∗ that maximizes between-class variance:
σ2
b (t) = ω0(t) ω1(t) (µ0(t) − µ1(t))2
where:
• ω0, ω1 are probabilities (class weights),
• µ0, µ1 are class means.
This works best when the histogram is close to bimodal (two peaks).
7
0.4.3 When Otsu Works / Fails
Works well when:
• near-bimodal histogram,
• relatively uniform illumination,
• moderate noise.
May fail when:
• non-uniform lighting (shadows/gradients),
• strong overlap between foreground and background intensities,
• object occupies a very tiny portion of the image,
• histogram not bimodal.
0.4.4 NumPy Implementation
def otsu_threshold_np ( img_u8 : np. ndarray ) -> int :
"""
Returns Otsu threshold t* in [0..255].
Implementation based on between -class variance .
"""
hist = np. bincount ( img_u8 . ravel () , minlength =256) . astype (np. float64
)
N = hist . sum ()
if N == 0:
return 0
p = hist / N
omega = np. cumsum (p)
mu = np. cumsum (p * np. arange (256) )
mu_T = mu [ -1]
eps = 1e -12
w0 = omega
w1 = 1.0 - omega
mu0 = mu / (w0 + eps)
mu1 = ( mu_T - mu) / (w1 + eps)
sigma_b2 = w0 * w1 * (mu0 - mu1) ** 2
t_star = int (np. argmax ( sigma_b2 ))
return t_star
def otsu_binarize_np ( img_u8 : np. ndarray ):
t = otsu_threshold_np ( img_u8 )
bin_img = threshold_binary_np (img_u8 , t)
return t, bin_img

PART 2. INTRODUCTION TO 2D CONVOLUTION
0.6 Introduction
Unlike Part 1, convolution is a neighborhood-based operation: each output pixel depends
on a local window around the input pixel. This concept is fundamental for:
• smoothing / denoising,
• sharpening,
• edge detection,
• and deep learning (CNNs).
0.7 Theory (Short)
Given an image x and a kernel K of size kH × kW , 2D convolution is:
y[i, j] =
kH −1∑
u=0
kW −1∑
v=0
K[u, v] · x[i + u − ∆h, j + v − ∆w]
where ∆h = ⌊kH/2⌋ and ∆w = ⌊kW/2⌋ for odd kernels.
0.8 Padding Modes: V ALID vs SAME
• V ALID (no padding): output is smaller; border pixels are not fully covered.
• SAME (with padding): output has same size as input; border is handled by padding.
Why padding matters: Without padding, border information is dropped. With
padding, you preserve spatial dimensions but must define what values exist beyond borders
(zero, reflect, replicate).
0.9 NumPy Convolution: No Padding (V ALID)
Workflow:
1. Convert to float for computation.
2. Flip kernel (true convolution vs correlation).
3. Slide kernel over all valid positions.
4. Compute weighted sum to produce output.
5. Normalize/clip for display.
def conv2d_valid_np ( img_u8 : np. ndarray , kernel : np. ndarray ) -> np.
ndarray :
"""
2D convolution without padding (VALID ).
Output size : (H-kH +1, W-kW +1) .
Returns uint8 for visualization ( normalized ).
10
"""
img = img_u8 . astype (np. float32 )
k = kernel . astype (np. float32 )
H, W = img. shape
kH , kW = k. shape
outH , outW = H - kH + 1, W - kW + 1
if outH <= 0 or outW <= 0:
raise ValueError ( " Kernel is larger than image for VALID
convolution .")
k_flip = np. flipud (np. fliplr (k))
out = np. zeros (( outH , outW ), dtype =np. float32 )
for i in range ( outH ):
for j in range ( outW ):
patch = img[i:i+kH , j:j+kW]
out[i, j] = np. sum ( patch * k_flip )
# normalize to [0 ,1] for visualization
mn , mx = out. min () , out. max ()
if mx - mn > 1e -6:
out01 = (out - mn) / (mx - mn)
else :
out01 = np. clip (out , 0, 255) / 255.0
return to_uint8 ( out01 )
0.10 NumPy Convolution: With Padding (SAME)
Padding types:
• zero: pad with zeros (may create dark borders)
• reflect: reflect image at the border
• edge: replicate edge values
def pad_image_np (img: np. ndarray , pad_h : int , pad_w : int , mode : str = "
zero ") -> np. ndarray :
"""
mode :
- 'zero ': constant 0 padding
- 'reflect ': reflect padding
- 'edge ': replicate border pixels
"""

return np.pad(img , (( pad_h , pad_h ), (pad_w , pad_w )), mode = "
constant ", constant_values =0)
if mode == " reflect ":
return np.pad(img , (( pad_h , pad_h ), (pad_w , pad_w )), mode = "
reflect ")

if mode == "edge ":
return np.pad(img , (( pad_h , pad_h ), (pad_w , pad_w )), mode = "edge
")

def conv2d_same_np ( img_u8 : np. ndarray , kernel : np. ndarray , padding_mode
: str = "zero ") -> np. ndarray :
"""
2D convolution with padding (SAME ): output size equals input size .
Kernel should be odd - sized (3 x3 , 5x5 ,...) .
Returns uint8 for visualization ( normalized ).
"""
img = img_u8 . astype (np. float32 )
k = kernel . astype (np. float32 )
kH , kW = k. shape
if kH % 2 == 0 or kW % 2 == 0:
raise ValueError ( "For SAME , kernel size should be odd (e.g., 3
x3 , 5x5 ).")
pad_h , pad_w = kH // 2, kW // 2
img_pad = pad_image_np (img , pad_h , pad_w , mode = padding_mode )
H, W = img. shape
k_flip = np. flipud (np. fliplr (k))
out = np. zeros ((H, W), dtype =np. float32 )
for i in range (H):
for j in range (W):
patch = img_pad [i:i+kH , j:j+kW]
out[i, j] = np. sum ( patch * k_flip )
mn , mx = out. min () , out. max ()
if mx - mn > 1e -6:
out01 = (out - mn) / (mx - mn)
else :
out01 = np. clip (out , 0, 255) / 255.0
return to_uint8 ( out01 )
